# 卓球オートスコア

卓球の試合動画をアップロードすると、ラリーの「動き」（フレーム差分）と「音」（打球音・歓声などの音量エネルギー）を解析し、ラリーが途切れたタイミング＝得点が入ったと思われるタイミングを自動で検出するブラウザ完結の採点支援ツール。サーバーには何も送信されず、すべてブラウザ内で処理されます。

ラリー検出は `<video>` 要素のネイティブ（ハードウェア）デコードと Web Audio API で行い、動画ファイル全体をメモリに読み込むことはありません。これにより、大きい動画（iPhoneのHEVC/Dolby Visionなど数百MB〜）でもタブがクラッシュ・フリーズしにくくなっています。スコアボードを焼き込んだ動画の書き出しのみ `ffmpeg.wasm` を使用します（この工程は動画全体をメモリに読み込むため、非常に大きい動画では時間がかかったりメモリ不足になる場合があります）。

## できること

1. 動画内の動き・音の解析から、ラリーの区切り（＝得点候補）を自動検出
2. 検出した候補をひとつずつ動画で確認しながら、どちらの得点かをワンクリック（またはキーボードショートカット）で確定
3. 卓球の公式ルール（11点先取・2点差、10-10からはデュース、サーブは2点ごとに交代／デュース以降は1点ごと）に沿ってスコア・ゲームカウント・サーブ権を自動計算
4. AIが見逃したラリーは、再生位置を合わせて手動で得点を追加可能。誤検出は「得点ではない」として除外、確定済みの判定もいつでも取り消し可能
5. 得点履歴をJSON/CSVでダウンロード
6. スコアボードを焼き込んだ動画（mp4）を書き出してダウンロード

**注意**：完全自動でボールやラリーの勝敗を100%正確に判定することは非常に難易度が高いため、本ツールは「AIがラリーの区切りを提案し、人が確認・修正する」半自動方式を採用しています。検出された候補は必ず動画で確認してください。

## 開発

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # 本番ビルド
```

`npm install` 時に `postinstall` で `@ffmpeg/core`（シングルスレッド版）を `public/ffmpeg/` へコピーし、外部CDNなしで自前ホストします（`public/ffmpeg/` はgit管理外）。`public/Anton.ttf` はスコアボード焼き込み用フォント（Anton, OFL-1.1）。焼き込み動画の選手名表示はこのフォントの対応範囲内（半角英数字）に限られます。
