# 卓球レッスンノート

生徒さん専用の、卓球レッスン記録アプリです。

- **① レッスンのまとめ**：レッスンの終わりにコーチが話す（またはテキストで書く）「今日教えたこと」を、AIが生徒向けの文章に自動でまとめます。
- **② 意識するポイント**：「練習や試合でいつも意識すべきポイント」をAIが管理。回を重ねるたびに、これまでの内容を踏まえて練り直され、その生徒専用にカスタマイズされていきます。
- **③ 試合結果**：生徒本人が、自分専用のページから試合結果と反省点・感想を記録できます。

データはあなた自身の**Googleスプレッドシート**に保存されます（「レッスン売上入力」アプリと同じく、専用サーバーは使わずブラウザから直接読み書きします）。スプレッドシートなので、中身はいつでもGoogleスプレッドシートの画面でそのまま確認・検索できます。

## しくみ

- **コーチ画面**：あなたのGoogleアカウントでログインし、スプレッドシートに直接読み書きします（レッスン売上入力アプリと同じ方式）。
- **生徒ページ**（個別リンク、ログイン不要）：スプレッドシート自体は非公開のまま、間に**Google Apps Script**という小さな仲介スクリプトを1つ挟みます。生徒がページを開くと、このスクリプトが「その生徒IDのデータだけ」を返し、試合結果の追記もこのスクリプト経由でのみ行われます。スプレッドシートを誰にでも見える状態で公開する必要はありません。

## セットアップ（初回のみ）

### 1. Google OAuth クライアントIDを用意する

**「レッスン売上入力」アプリで使っているクライアントIDが既にあれば、それをそのまま使い回せます**（同じGoogleアカウント・同じ公開ドメインなら、新しく作る必要はありません）。手順3でそのIDを入力してください。

まだ無い場合は、新しく作成します（無料）。

1. [Google Cloud Console](https://console.cloud.google.com/) で新しいプロジェクトを作成（または既存のものを使用）
2. 「APIとサービス」→「ライブラリ」で **Google Sheets API** を有効化
3. 「APIとサービス」→「OAuth同意画面」を設定
   - User Type は「外部」でOK（個人利用なら「テスティング」のままで問題ありません）
   - 「テストユーザー」に自分の Google アカウントを追加
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類：**ウェブ アプリケーション**
   - 「承認済みの JavaScript 生成元」に、公開するURLのオリジンを追加（例: `https://kazuhideyukinori-cpu.github.io`。ローカルで試す場合は `http://localhost:5173` も）
   - 作成後に表示される「クライアントID」をコピー

### 2. レッスン記録用のスプレッドシートを作成する

新しいGoogleスプレッドシートを1つ作成し、以下の3つのシート（タブ）を作ります。**シート名・見出し行の文言は下の通りに正確に入力してください**（アプリとApps Scriptがこの文言で列を判別します）。

**「生徒」タブ**（1行目に見出し）
```
ID	名前	意識するポイント	作成日時	更新日時
```

**「レッスン」タブ**
```
ID	生徒ID	日時	書き起こし	まとめ
```

**「試合結果」タブ**
```
ID	生徒ID	記録日時	試合日	対戦相手	結果	反省・感想
```

このスプレッドシートは**共有設定はそのまま（非公開）でOK**です。生徒に共有する必要はありません。

URLの `/d/` と `/edit` の間にある文字列（スプレッドシートID）を控えておいてください。

### 3. Apps Script を導入する（生徒ページ用のAPI）

1. 作成したスプレッドシートを開き、「拡張機能」→「Apps Script」を開く
2. 最初から入っているコード（`function myFunction() {}` など）を全部消して、このリポジトリの [`apps-script/Code.gs`](./apps-script/Code.gs) の中身をそのまま貼り付ける
3. 保存（フロッピーアイコン）
4. 右上の「デプロイ」→「新しいデプロイ」
   - 種類の選択（⚙アイコン）→「ウェブアプリ」
   - 「次のユーザーとして実行」：**自分**
   - 「アクセスできるユーザー」：**全員**
   - 「デプロイ」をクリック
   - 初回は権限の承認を求められるので、自分のGoogleアカウントで許可（「このアプリはGoogleで確認されていません」と出た場合は「詳細」→「(プロジェクト名)に移動」→許可、でOKです。自分専用のスクリプトなので安全です）
5. 表示された「ウェブアプリ」のURL（`https://script.google.com/macros/s/.../exec` の形式）をコピー

**注意**：Apps Scriptのコードを後から書き換えた場合、保存しただけでは公開中のURLには反映されません。「デプロイ」→「デプロイを管理」→ 編集（鉛筆アイコン）→ バージョン「新バージョン」→「デプロイ」で更新してください。

### 4. Apps Script のURLをアプリに設定する

`src/public-config.ts` を開き、手順3でコピーしたURLを貼り付けてください。

```ts
export const APPS_SCRIPT_URL = "https://script.google.com/macros/s/xxxxxxxx/exec";
```

このURLは生徒ページから直接呼び出す公開エンドポイントであり秘密情報ではないため、そのままコミットして問題ありません（アクセス制御はURLを隠すことではなく、Apps Script側で「渡された生徒IDのデータしか返さない」ようにすることで行っています）。

### 5. Anthropic APIキーを取得する

1. [Anthropic Console](https://console.anthropic.com/) でAPIキーを発行
2. アプリを開き、右上の「⚙ 接続設定」で以下を入力して保存
   - **Google OAuth クライアントID**（手順1）
   - **スプレッドシートID**（手順2）
   - **Anthropic APIキー**
   - AIモデル名は初期値 `claude-sonnet-5` のままで問題ありません
   - これらはコーチ本人の端末のブラウザにのみ保存されます

### 6. Googleでログイン

アプリを開いて「Googleでログイン」→ 手順1のクライアントIDを発行したGoogleアカウント（スプレッドシートの所有者）でログインすれば、生徒の追加・レッスン記録ができるようになります。

## 使い方の流れ

1. コーチ画面で生徒を追加し、発行された個別リンクを生徒に共有（LINEなどで送るだけでOK、ログイン不要）
2. レッスン後、その生徒を選んで「🎙 録音開始」→ 今日教えたことを話す → 「⏹ 録音停止」
   - （音声認識非対応ブラウザの場合は、テキストエリアに直接入力）
3. 「AIでまとめる」→ 内容を確認・必要なら編集 →「保存する」
   - 「今日のまとめ」がレッスン履歴に追加され、「意識するポイント」も最新版に更新されます
4. 生徒は個別リンクを開けば、いつでも「意識するポイント」とレッスン履歴を見返せます
5. 生徒は試合の後、同じページから試合結果と反省点・感想を記録できます
6. 中身を見たくなったら、いつでもGoogleスプレッドシートを直接開いて確認できます

## 開発

```bash
npm install
npm run dev      # ローカルで確認
npm run build    # 本番ビルド（dist/ に出力）
```

## 注意点

- Anthropic APIの利用料金はAPIキーの発行者（コーチ）に発生します。
- 音声認識は Web Speech API を使用しており、Chrome など対応ブラウザでのみ動作します（対応状況はブラウザに依存します）。非対応の場合はテキスト入力で代用できます。
- 生徒ページ用のApps Script URLは公開エンドポイントです。生徒IDが第三者に漏れた場合、その生徒のデータ閲覧・試合結果の追記が可能になってしまうため、個別リンクは本人以外に共有しないよう伝えてください。
