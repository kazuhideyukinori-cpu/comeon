# レッスン売上入力

「経費帳」スプレッドシートの **毎月のレッスン売上シート**（`レッスン売上4月` `レッスン売上5月` …）に、フォームから直接1行を追記するための小さなアプリです。ブラウザだけで完結し、サーバーは経由せず、あなたの Google アカウントの権限で Google Sheets API に直接書き込みます。

## できること

- 月ごとのシート（`レッスン売上◯月`）を選んで、そのまま1行追加
- 「場所」を選ぶと、直近で使った「場所代」「出発駅」「到着駅」「往復・片道」を自動入力（手で書き換えれば上書きされます）
- 「場所」「人」「出発駅」「到着駅」は過去の入力から候補が出るオートコンプリート
- 「交通費」は、過去の入力から同じ区間（出発駅・到着駅）の運賃を学習し、片道／往復の選択に応じて自動計算（手で書き換えれば上書きされます。まだ使ったことのない区間は自動計算されません）
- これらの自動入力・オートコンプリートは、今開いている月のシートだけでなく **スプレッドシート内の「レッスン売上◯月」タブすべて** の履歴から学習します。新しく月のシートを作ったばかりで中身が空でも、過去の月の場所・駅・運賃をもとに自動入力されます
- 同じ日付が続く行は日付欄を空欄にし、日付が変わったら区切りの空白行を1行挟む、というシートの既存の書き方に合わせて自動調整
- 追加した行は、直前の行の書式（通貨表示・プルダウンなど）をコピーして体裁を保つ
- 「＋ 新しい月のシートを作成」で、既存の月シートを複製して次の月のタブを作成（データは空にして書式だけ引き継ぐ）

## セットアップ（初回のみ）

Google スプレッドシートに書き込むには、Google Cloud で自分用の OAuth クライアントIDを作る必要があります（無料）。

1. [Google Cloud Console](https://console.cloud.google.com/) で新しいプロジェクトを作成（または既存のものを使用）
2. 「APIとサービス」→「ライブラリ」で **Google Sheets API** を有効化
3. 「APIとサービス」→「OAuth同意画面」を設定
   - User Type は「外部」でOK（個人利用なら「テスティング」のままで問題ありません）
   - スコープの追加は不要（アプリ側で `spreadsheets` スコープを要求します）
   - 「テストユーザー」に自分の Google アカウント（このスプレッドシートを持っているアカウント）を追加
   - ※テスティング状態のままだと、約7日でログインし直しが必要になることがあります。気にならなければそのままで大丈夫です
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
   - アプリケーションの種類：**ウェブ アプリケーション**
   - 「承認済みの JavaScript 生成元」に、このアプリを公開する URL のオリジンを追加
     - 例: `https://kazuhideyukinori-cpu.github.io`（GitHub Pages で公開する場合）
     - ローカルで試す場合は `http://localhost:5173` も追加すると便利です
   - 作成後に表示される「クライアントID」（`xxxxx.apps.googleusercontent.com` の形式）をコピー
5. アプリを開き、右上の「⚙ 接続設定」を開いて
   - **Google OAuth クライアントID** に手順4のIDを貼り付け
   - **スプレッドシートID** はスプレッドシートのURLの `/d/` と `/edit` の間の文字列（初期値は今回対象のシートIDが入っています）
   - 「保存」を押す
6. 「Googleでログイン」→ 権限の確認画面が出るので許可

設定は自分のブラウザの localStorage にのみ保存され、どこにも送信されません。

## 交通費の自動計算について（任意設定）

交通費は基本、**過去にこのシートへ入力した実績（区間ごとの運賃）を学習して自動計算**します。追加費用はかかりません。

まだ一度も使ったことがない区間（初めて通る駅の組み合わせ）は学習データがないため自動計算されませんが、以下の設定をすると、その場合だけ外部の運賃API（駅すぱあと API、Suica等のIC運賃）に自動で問い合わせて埋めるようになります。**任意の追加機能なので、設定しなくても既存の学習ベースの自動計算はそのまま動きます。**

### 1. 駅すぱあと API のアクセスキーを購入する

Amazon.co.jp で「**駅すぱあと API for Amazon**」を購入します（5,000リクエストで ¥5,500 ほど、個人利用も想定された価格帯です）。購入後、確認ページからアクセスキーを取得できます。

- リクエスト1回＝おおよそ1〜1.1円の従量制です。このアプリは「履歴にない区間だけ」問い合わせる作りなので、消費は最小限に抑えられます

### 2. Cloudflare Workers で中継サーバーを立てる（無料）

アクセスキーをブラウザ側に直接埋め込むと誰でも使えて課金され放題になってしまうため、キーを隠しつつCORSを許可する小さな中継サーバー（Cloudflare Workers）を挟みます。

1. [Cloudflare](https://dash.cloudflare.com/sign-up) の無料アカウントを作成
2. ダッシュボードで「Workers & Pages」→「Create」→「Create Worker」
3. エディタの中身を全部削除し、このリポジトリの [`lesson-sales-entry/cloudflare-worker/fare-proxy.js`](./cloudflare-worker/fare-proxy.js) の内容を貼り付けて「Deploy」
4. デプロイ後、Worker の「Settings」→「Variables」→「Add variable」で
   - 変数名: `EKISPERT_ACCESS_KEY`
   - 値: 手順1で取得したアクセスキー
   - 「Encrypt」（Secret扱い）にチェックして保存
5. Worker のURL（`https://xxxxx.workers.dev` の形式）をコピー

### 3. アプリに設定する

アプリの「⚙ 接続設定」→「運賃検索プロキシURL」に、手順2-5でコピーしたURLを貼り付けて保存すれば完了です。

### 注意

- 駅すぱあとの正式なAPI仕様書は開発環境から参照できなかったため、`fare-proxy.js` のパラメータ（`condition=ic` など）は仕様の要約情報をもとにした最善の推測です。実際のアクセスキーで試して運賃が正しく取れない場合は、エラーメッセージ（`raw` に生レスポンスが含まれます）を教えてもらえれば調整します
- 出発駅・到着駅は駅すぱあとの駅マスタに登録されている表記（「〇〇駅」ではなく「〇〇」など）と一致している必要がある場合があります。うまく検索できない駅名があれば教えてください

## 開発

```bash
npm install
npm run dev      # ローカルで確認
npm run build    # 本番ビルド（dist/ に出力）
```

## 注意点

- シート名は `レッスン売上{数字}月` という命名を前提にしています（例: `レッスン売上7月`）。
- 列の並びはシートの1行目（見出し行）を毎回読み取って対応させているので、列の順番が多少変わっても動きますが、見出しの文言（`日付` `場所` `人` `時間` `売上` `場所代` `交通費` `出発駅` `到着駅` `往復・片道`）は変えないでください。
- 数字系の列（時間・売上・場所代・交通費）はいずれも半角数値として書き込まれます。
